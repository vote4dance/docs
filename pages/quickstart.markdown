# Quickstart

This flow gets a new organizer from zero to a runnable event.

## 1. Create an event in Manager

In `Manager`, create a new event and complete the 4 setup steps:

1. Federation
2. Event details
3. Location and timezone
4. Billing or practice setup

For practice events, choose a bootstrap setup if you want automatic defaults.

## 2. Use practice bootstrap when testing

Practice bootstrap can automatically create baseline objects, including:

- A main floor
- A judging panel
- Default schedule structure
- A starter screen
- Default cafe products
- Practice classes, generated rounds, and random teams

Use this mode to learn the full workflow before production events.

## 3. Build your event structure

In `Manager Guide`, do this order:

1. Event setup: competitions, floors, users, schedule
2. Competition setup: classes, rounds, teams, judges
3. Registration and check-in setup

## 4. Dry run before event day

Run a full rehearsal with your team:

1. Set one round to `Ready`
2. Judge in the judging app
3. Close, confirm, and publish results in Manager
4. Verify screens, speaker flow, and check-in behavior

## 5. Live operations sequence

For each round, use this lifecycle:

1. `Open`
2. `Ready`
3. `Closed`
4. `Confirmed`
5. `Published`

When the event is finished, run post-event tasks from [Post-event and exports](/manager-guide/post-event/).
