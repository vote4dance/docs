# Federation Rankings

This section explains how federation rankings work in Vote4Dance.

## Start here

Choose the page that matches what you are doing:

1. [Federation admin](/federation-rankings/federation-admin/)  
   Use this if you are creating, reviewing, or correcting rankings.
2. [Public rankings](/federation-rankings/public/)  
   Use this if you want to understand how published rankings are shown to dancers, teams, and visitors.

## Core rule

Every ranking belongs to one division.

This means:

- a ranking must be created inside a division
- a ranking cannot exist without a division
- ranking pages are always opened from a division context
- public ranking links always include the division

## What a ranking contains

A federation ranking combines:

- one `division`
- one ranking profile
- one or more included federation classes
- the resolved federation ranking license policy
- published standings
- optional manual corrections

## Recommended workflow

Follow this order when working with federation rankings:

1. Confirm the division structure is correct
2. Create the ranking inside the correct division
3. Add the federation classes that should feed the ranking
4. Confirm the resolved ranking license policy matches what the federation expects
5. Review the published standings
6. Apply manual corrections if needed
7. Verify the public ranking page in the same division

## Common mistakes

Most ranking problems come from one of these:

- the ranking was created in the wrong division
- the needed federation classes were not added to the ranking
- the ranking license policy was assumed instead of checked
- someone expected the ranking to be federation-wide instead of division-specific
- a manual correction was applied to the wrong division ranking
