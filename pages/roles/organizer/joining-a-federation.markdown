# Organizer joining a federation

An organizer usually aligns to a federation through the organization and event settings, not as an isolated personal action.

## What this step is for

This step ensures your event is configured under the correct federation rules before registration opens.

## Before you start

Confirm these prerequisites:

- Your organizer account has Manager access
- Your organization context is correct
- Federation contact/admin is known

## Where to go

1. Open **Manager**
2. Open your event
3. Go to **Overview** or **Federation settings** in event setup

You may also need to check with federation admin tools:

- [Federation admin](/federation-licenses/federation-admin/)

## Step-by-step

### Step 1: Confirm governing federation

Before editing event structure, explicitly confirm:

- federation name
- season/year
- whether this event is ranking-eligible

Keep this confirmation written (email or doc) to avoid disputes later.

### Step 2: Verify organization approval status

Check whether your organization is approved under that federation.

If approval is pending:

- do not open registration yet
- escalate to federation admin

### Step 3: Align event rules to federation policy

Review and apply federation requirements for:

- required licenses
- allowed classes/divisions
- age transitions
- class transfers
- rankings eligibility

### Step 4: Validate class and license consistency

In Manager, compare:

1. configured classes in event
2. federation-approved class structure
3. license requirements for each class

If mismatched, fix before registration opens.

### Step 5: Confirm ranking path

If federation rankings are expected:

1. Verify results from this event are eligible
2. Verify required statuses/workflow for publication
3. Verify any federation-specific export/sync steps

### Step 6: Final pre-registration check

Before opening registration, complete this quick check:

- Federation is correct
- Organization is approved
- Classes match federation model
- License policy is clear and visible
- Ranking behavior is confirmed

## Common blockers and fixes

### "Event was configured before federation rules were clear"

Fix:

1. Freeze registration opening
2. Compare classes/licenses against federation policy
3. Correct setup and document changes

### "Organization is not approved by federation"

Fix:

1. Contact federation admin
2. Submit required approval documentation
3. Re-check status before continuing

### "Dancers expect different class structure"

Fix:

1. Publish corrected class map
2. Communicate updates before registration
3. Avoid silent changes after registrations begin

## What success looks like

Your event structure, eligibility checks, and ranking expectations fully match the governing federation.

## Best practical check

Before registration opens, compare and sign off these three items:

1. federation expected by participants
2. classes and divisions in Manager
3. enforced license and eligibility policy

## Next step

Continue to [Registration for event](/organizer/registration-for-event/) once federation alignment is clear.
