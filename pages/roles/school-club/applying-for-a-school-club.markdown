# Applying for a school or club

At organization level, this step means making your school/club profile clear, searchable, and ready for members to join without confusion.

## What this step is for

You are preparing your organization identity so dancers, coaches, and admins can join the correct school/club and use it in licensing and event workflows.

## Before you start

Prepare:

- official organization name
- federation affiliation status
- at least two admin users
- public contact email

## Step-by-step

### Step 1: Complete organization profile

In organization settings, fill:

- name (exact official form)
- type (school, studio, club)
- location
- contact email
- short description

Use the name members already recognize to avoid duplicate requests.

### Step 2: Configure admin ownership

Ensure at least two trusted admins are active:

1. open **Members/Users**
2. add backup admin
3. verify both can edit organization settings

### Step 3: Make membership path clear

Decide how members join:

- invite-only
- request-to-join
- federation-linked auto path (if supported)

Publish one clear instruction for members (where to click and who approves).

### Step 4: Validate discoverability

Run a quick test using a non-admin account:

1. search for your organization by name
2. verify only one correct profile appears
3. submit join request or accept invite

If users can find multiple similar names, improve naming/description.

### Step 5: Confirm readiness before inviting members

Before broad invitations:

- admin roles confirmed
- profile complete
- join workflow tested
- federation link status known

## Common blockers and fixes

### "Members join the wrong organization"

Fix:

1. rename profile to clearer official name
2. add location and federation details
3. publish exact join instructions

### "No admin available to approve requests"

Fix:

1. add backup admin immediately
2. verify notifications are enabled
3. set response SLA for membership approvals

### "Duplicate school profiles exist"

Fix:

1. choose one canonical profile
2. migrate active members to canonical profile
3. archive or hide duplicate profile

## What success looks like

Members reliably find and join the correct school/club with minimal support help, and all downstream licensing/registration flows use the correct organization context.
